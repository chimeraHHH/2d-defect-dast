# 二维材料缺陷预测平台 - 晶体结构文件格式说明

本平台后端使用了 [ASE (Atomic Simulation Environment)](https://wiki.fysik.dtu.dk/ase/) 的 `ase.io.read()` 方法来解析用户上传的晶体结构文件。由于该方法具备自动推断文件格式的强大功能，本平台能够支持绝大多数主流的晶体结构和分子结构文件格式。

以下是平台主要推荐和支持的文件格式：

## 1. VASP 格式 (推荐)
- **POSCAR** / **CONTCAR**
  - **说明**：这是 VASP（Vienna Ab initio Simulation Package）软件中用于定义晶胞参数和原子坐标的标准文件格式。
  - **特点**：对于周期性边界条件（PBC）的二维材料结构描述非常准确，是材料计算领域最常用的格式之一。
  - **示例文件**：平台测试集中的 `POSCAR_hBN` 就是典型的 VASP 格式文件。

## 2. CIF 格式 (.cif) (推荐)
- **Crystallographic Information File**
  - **说明**：国际晶体学联合会（IUCr）发布的晶体学信息文件标准，用于交换和归档晶体结构数据。
  - **特点**：包含详细的晶格参数、空间群信息以及分数坐标，支持非常广泛。
  - **示例文件**：平台测试集中的 `MoS2_monolayer.cif` 和 `graphene.cif`。

## 3. XYZ 格式 (.xyz, .extxyz)
- **XYZ / Extended XYZ**
  - **说明**：一种简单的基于笛卡尔坐标的文件格式。第一行是原子总数，第二行是注释（Extended XYZ 会在这里存储晶胞参数等信息），后面是各原子的元素符号及坐标。
  - **特点**：标准 XYZ 格式通常不包含晶格参数，因此在处理需要周期性边界条件的二维材料时，**建议使用 Extended XYZ 格式**（在注释行包含 `Lattice="..."` 属性），以确保晶胞信息不丢失。

## 4. 其他支持的格式
得益于 ASE 库的兼容性，平台还隐式支持以下格式（只要文件后缀名或内容格式符合标准）：
- **PDB (.pdb)**：Protein Data Bank 格式，常用于生物大分子，也可用于纳米材料。
- **Quantum Espresso (.in, .pwi)**：QE 的输入文件格式。
- **LAMMPS 数据文件 (.data)**：用于分子动力学模拟的结构文件。
- **Gaussian (.gjf, .com)** 等。

## 💎 模型所需的晶体核心信息

为了使后端的深度学习图神经网络能够准确预测缺陷形成能，上传的晶体结构文件必须包含以下核心信息。如果解析时缺失这些信息，可能会导致特征提取失败：

1. **原子种类 (Atomic Elements / Numbers)**
   - **说明**：结构中每个原子的具体元素类型（如 B, N, C, Mo, S 等）。
   - **用途**：模型会将不同的元素种类映射为预加载的原子特征张量（Atom Features），作为图结构中各个节点（原子）的初始特征。

2. **原子坐标 (Atomic Positions / Coordinates)**
   - **说明**：晶胞中每个原子的三维空间位置（支持笛卡尔坐标或分数坐标）。
   - **用途**：用于构建晶体的空间拓扑图，计算原子之间的相对距离（Edge Distances）以及相邻化学键构成的空间三体夹角（Angles），从而精确捕捉缺陷导致的局部几何畸变。

3. **晶胞参数 (Lattice Parameters / Cell Vectors)**
   - **说明**：定义材料周期性边界条件（PBC）的晶格基矢（通常表现为 3x3 的矩阵，或晶格常数 a, b, c 及其夹角）。
   - **用途**：由于二维材料是周期性延展的，模型在计算原子间距时必须依赖晶胞参数来应用**最小镜像约定 (Minimum Image Convention, MIC)**。这确保了位于晶胞边界处的原子能够跨越边界，与相邻周期镜像中的真实邻居正确相连。如果缺失此信息（如使用基础的 `.xyz` 文件），晶体边界处的图连通性将被破坏，导致预测结果失真。

---

## 💡 上传建议

1. **优先使用 POSCAR 或 CIF**：为了确保深度学习模型能够准确提取原子的周期性边界条件（PBC）和距离矩阵（基于 Minimum Image Convention），请优先上传包含完整晶胞参数（Cell parameters）的 `POSCAR` 或 `.cif` 文件。
2. **文件名后缀**：尽量保持标准的文件后缀（如 `.cif`, `.xyz`）或标准文件名（如 `POSCAR`），这样有助于后端 `ase.io.read()` 更快、更准确地自动识别文件类型。
